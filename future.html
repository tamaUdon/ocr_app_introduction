画像認識の前処理をふやす(2019/03/18実装済)

画像にゆがみがあると、その後の文字認識の精度が下がる。
画像のゆがみを検出し(テキスト検出領域の座標などから割り出す)、画像を回転させる処理を挟む。
最小二乗法で文字列の座標から回帰直線の傾きを求め、画像を傾きの角度だけ回転させた。

複数行認識を可能にする(2019/03/27実装済)

現状では、複数行にわたる文字列を回転できない。
VBGMM (Variational Bayesian Gaussian Mixture Model) により列ごとにクラスタリングすることで、
改行を認識できる可能性がある。(現在、クラスタ化の実装まで完了)
VBGMMは、ラベルデータなし、分割するクラスタの数が不明でもクラスタ化できる。 チューニング次第で空白も認識できるか試す。

→各文字の中点座標のみをとりだしてクラスタリングをしたが、3行以上になると精度が出ない。
チューニングの問題もあるかもしれないが、各文字1点のみではデータが少ないかもしれないと思い、
AKAZEによる特徴点検出を試している。特徴点が集まっているところをクラスタリングするようにすれば、精度が上がるかもしれない。

→2019/03/27
特徴点検出してからクラスタリングすることで、複数行にわたる文字列を含む画像のゆがみを補正することができた。
→2019/04/22
単語区切りでスペースを入れられるようにした。OpenCVでdilation処理を施してからfindContours関数を使うと、
スペース区切りで矩形を取得することができる。

自然言語処理

MeCab, OpenNLPなどのライブラリを使用した自然言語処理で精度向上を図る。
英字・数字を分類できるか要検討。
→2019/04/22 Microsoft Azure の Bing Spell Check API を使用し、予測精度を上げることにした。

英数同時認識(2019/04/22実装済)

現状では、数字か英字かどちらか一方のみで構成された文字列しか認識できない。
そこで、英字予測モデルと数字予測モデルで計2回予測し、平均予測確率が高い方を英字/数字と判断する方法を考えている。
(処理が重いはずなので代替え案も考える)

目的を明確にすることで、機能の実装もやりやすくなるのではないだろうか。

→2019/04/05
Androidアプリとして公開する予定だったが、OCR機能を作るので精いっぱいなので、
webアプリにすることを検討中。htmlとjavaScriptでwebブラウザからカメラを起動できるらしい。
webから直接APIに画像を投げれば、トリミングの必要もないかもしれない。
(Androidは送信できる画像の大きさの上限が決まっており、画像を小さくしなければならなかった)
